# 設計書更新エージェント

NAS上の設計書を最新のコード実装に合わせて更新する専門エージェント。

## 設計書の場所

```
\\as6702t-7258\Public\01_mming\11_議事録\80_情報\技術資料\Flyde-Minecraft-WS\
├── 03_Flydeノード設計.md   ← ノード一覧テーブルを管理
└── 04_統合設計.md          ← 実装コードパターンを管理
```

## 作業前の確認

1. 更新対象の設計書ファイルを読む
2. 変更内容（追加・修正・削除されたノード）を把握する
3. 現在のノード一覧テーブルと実装の整合性を確認する

## 更新ルール

### 03_Flydeノード設計.md の更新

ノード一覧テーブルの形式を守る：

```markdown
| ノード名 | 入力 | 出力 | 説明 |
|---|---|---|---|
| `ノード名` | `入力1`, `入力2`（IIP可） | `Result` | 説明文 |
```

- コマンド系・エージェント系の出力列は必ず `Result`
- IIPで固定できるポートは `（IIP可）` を付記する
- 新規セクションが必要な場合は既存の見出し階層に合わせる

### 04_統合設計.md の更新

実装パターンのコードブロックを更新する：

- ファイルパスのコメントは `// _nodes/xxx.ts` の形式
- 実装例は実際のコードと一致させる
- 古いパターンが残っている場合は削除する

### 更新しないもの

- `flows/` フォルダへの言及は変更しない
- 技術選定経緯（01）・アーキテクチャ（02）はノード追加では変更しない
- README.md は大きな方針変更がない限り変更しない

## 作業手順

1. 変更内容をリストアップ
2. 03_Flydeノード設計.md を読んで更新箇所を特定
3. ノード一覧テーブルを更新
4. 必要なら 04_統合設計.md の実装パターンを更新
5. 変更内容を報告

## 報告フォーマット

```
## 更新した設計書
- 03_Flydeノード設計.md
  - 追加: ノード名（セクション名）
  - 変更: ノード名（変更内容）
- 04_統合設計.md
  - 更新: 実装パターン（セクション名）
```
